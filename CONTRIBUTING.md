# Contributing Guidelines

Thank you for your interest in contributing to my portfolio website! I welcome feedback, suggestions, and improvements to enhance this project. Below are the guidelines to help you get started.

## How to Contribute

### Reporting Issues
- If you find a bug or have a suggestion, please open an issue in the [GitHub Issues](https://github.com/yourusername/your-repo/issues) section.
- Provide a clear description, steps to reproduce, and any relevant details (e.g., browser or device used).

### Suggesting Features
- Propose new features or enhancements by opening an issue with the label "enhancement."
- Include a detailed explanation of the feature and its potential benefits.

### Submitting Changes
1. **Fork the Repository**:
   - Create your own copy of the repository by clicking the "Fork" button.
2. **Create a Branch**:
   - Use a descriptive name (e.g., `feature/add-contact-form`).
   - Run `git checkout -b branch-name`.
3. **Make Changes**:
   - Edit files (e.g., `index.html`, CSS, or JavaScript) and test locally.
   - Ensure your changes align with the existing style and functionality.
4. **Commit Changes**:
   - Run `git add .` and `git commit -m "Description of changes"`.
5. **Push to GitHub**:
   - Run `git push origin branch-name`.
6. **Submit a Pull Request (PR)**:
   - Open a PR from your fork to the main repository.
   - Describe your changes and why they’re needed.

### Code Standards
- Use consistent indentation (2 spaces) and follow the existing HTML, CSS, and JavaScript structure.
- Test your changes on multiple devices (desktop and mobile) to ensure responsiveness.
- Avoid breaking existing features (e.g., resume toggle, scroll behavior).
